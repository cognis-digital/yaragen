<a name="why"></a>
## Why yaragen?

Generate candidate YARA rules from sample files/strings — without standing up heavyweight infrastructure.

`yaragen` is single-purpose, scriptable, and self-hostable: point it at a target, get prioritized results in the format your workflow already speaks (table · JSON · SARIF), gate CI on it, and let agents drive it over MCP.

<a name="quick-start"></a>
## Quick start

```bash
pip install cognis-yaragen
yaragen --version
yaragen scan .                       # scan current project
yaragen scan . --format json         # machine-readable
yaragen scan . --fail-on high        # CI gate (non-zero exit)
```

<a name="ai-stack"></a>
## Use it from any AI stack

`yaragen` is interoperable with every popular way of using AI:

- **MCP server** — `yaragen mcp` (Claude Desktop, Cursor, Cognis.Studio, [uncensored-fleet](https://github.com/cognis-digital/uncensored-fleet))
- **OpenAI-compatible / JSON** — pipe `yaragen scan . --format json` into any agent or LLM
- **LangChain · CrewAI · AutoGen · LlamaIndex** — wrap the CLI/JSON as a tool in one line
- **CI / scripts** — exit codes + SARIF for non-AI pipelines
